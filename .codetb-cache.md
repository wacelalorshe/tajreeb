### 1. PROJECT OVERVIEW
**Type:** Full-stack IPTV portal for World Cup 2026 live streaming, match tracking, and video management.  
**Primary stack:** PHP backend (REST API, JSON file storage), vanilla JavaScript frontend (HLS.js player, PWA), TypeScript FFmpeg streaming services, MySQL database, Android WebView wrapper.  
**What it does:** Authenticates users, streams live TV channels and matches via HLS, manages video uploads (chunked), displays match schedules/scores, provides live chat, and supports PWA installation. The system uses JSON files for most data persistence with MySQL for structured data (users, channels, sessions).

### 6. MODIFICATION RULES
**Critical constraints:**
1. **JSON file locking:** All `data/*.json` files are read/written by multiple PHP scripts concurrently. Must use `flock()` or implement file-level locking to prevent corruption.
2. **Session integrity:** `active_sessions.json` is the single source of truth for authentication. Never modify session logic without verifying device-based concurrent session limits.
3. **Cache TTLs:** `scores_cache.json` (60s) and `matches_cache.json` must be respected. Clearing cache prematurely breaks live score updates.
4. **HLS proxy:** `proxy.php` must remain memory-efficient (streams segments, doesn't buffer entire files). Changing manifest rewriting logic can break stream delivery.
5. **Ched upload:** `video_upload.php` and `js/chunked-uploader.js` must stay in sync regarding chunk size, session IDs, and assembly logic.
6. **Auth guard:** Every protected HTML page (`live.html`, `tv.html`, `matches.html`, `profile.html`, `library.html`) calls `verifySession()` on load. New pages must follow this pattern.
7. **FFmpeg paths:** `hardware.php` and `bin/cmd.txt` assume FFmpeg is installed via Chocolatey. Changing installation method requires updating all FFmpeg references.
8. **Android WebView:** `MainActivity.java` loads the portal URL. Any URL structure changes must be reflected in the Android app's configuration.
9. ** tests:** `tests/` directory contains fast-check tests for username uniqueness and plan monotonicity. Changes to user/plan logic must pass these tests.
10. **RTL support:** Arabic UI elements (login, dashboard) use `dir="rtl"`. New UI components must respect RTL layout.
